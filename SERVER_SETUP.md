# Настройка на сервере

## Проблема: ветка addnews не найдена

Ветка `addnews` еще не была отправлена в удаленный репозиторий или не была получена на сервере.

## Решение

### Шаг 1: На локальной машине (Windows)

Выполните эти команды в терминале в папке проекта:

```bash
cd d:\xampp\htdocs\WP\wordpressportal

# Убедитесь, что вы в ветке addnews
git checkout addnews

# Добавьте все изменения
git add -A

# Создайте коммит
git commit -m "feat: добавлены пользовательские поля, синхронизация новостей, фильтры по тегам"

# Отправьте ветку в удаленный репозиторий
git push -u origin addnews
```

### Шаг 2: На сервере

После того, как ветка будет отправлена с локальной машины, на сервере выполните:

```bash
# Получите все ветки из удаленного репозитория
git fetch origin

# Проверьте доступные ветки
git branch -r

# Создайте локальную ветку addnews, отслеживающую удаленную
git checkout -b addnews origin/addnews

# Или если ветка уже существует удаленно:
git checkout addnews
```

### Альтернативный вариант: создать ветку на сервере напрямую

Если вы хотите создать ветку на сервере из текущего состояния:

```bash
# На сервере
cd /var/www/wordpress

# Создайте новую ветку из текущей
git checkout -b addnews

# Добавьте все изменения (если они есть)
git add -A

# Создайте коммит
git commit -m "feat: добавлены пользовательские поля, синхронизация новостей, фильтры по тегам"

# Отправьте в репозиторий
git push -u origin addnews
```

### Шаг 3: Проверка

После выполнения команд проверьте:

```bash
# На сервере
git branch  # должна показать ветку addnews с *
git status  # покажет статус
```

## Если ветка уже существует в удаленном репозитории

Если ветка уже была отправлена с локальной машины, но не видна на сервере:

```bash
# На сервере
git fetch --all
git branch -a  # покажет все ветки, включая удаленные
git checkout -b addnews origin/addnews
```
